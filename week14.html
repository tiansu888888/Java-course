Week 14: Group Project Demo 第 14 周：小组项目演示

Step-by-Step Guide 分步指南

1. Presentation Skills

A good software demo requires explaining the problem, demonstrating the solution, and reviewing the code architecture.

2. Code Review

Reviewing code helps identify bugs, ensure readability, and confirm that project requirements (like proper loop/method usage) are met.

1. 演示技巧

一个优秀的小组软件演示需要讲解问题痛点、展示最终解决方案，并复盘代码架构设计。

2. 代码审查

代码审查能够帮助发现潜在的 bug、确保代码可读性，并确认是否符合所有的项目规范（例如正确使用了循环与方法）。

Concept Visualization 概念可视化

Tips 提示

Speak clearly and be prepared to explain why you chose a specific loop or method. Confidence comes from knowing your code!

口齿清晰，并准备好解释你为什么选择使用特定循环或方法。信心来源于吃透你写的每一行代码！

Knowledge Check 知识自测

2. What does UI stand for? UI 代表什么？

3. Why is input validation important? 为什么输入验证如此重要？

4. Which Java feature is best for formatting console output beautifully? 哪种 Java 特性最适合将控制台输出美化格式化？

5. What should you do if your program crashes during a live demo? 如果在进行现场演示时程序突然崩溃，你应该怎么做？
